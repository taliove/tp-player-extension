# UI 优化设计文档 — Teleport RDP Web Player Chrome Extension

日期: 2026-03-30

## 概述

优化 Chrome 扩展的两个核心界面：会话审计列表页和 RDP 播放器页面。目标是让审核候选人机试过程更高效，突出关键信息（时长），增加快捷操作、录像缓存、笔记系统和播放历史。

## 优先级

1. 列表页优化
2. 播放器快捷键 + 动画
3. 录像缓存
4. 顶部菜单栏 + 播放历史
5. 右侧笔记侧边栏

---

## 1. 列表页 — 紧凑行卡片

### 改造方式

注入 CSS 隐藏原始表格（`#table-record` 或对应的 `<table>`），在其上方插入卡片列表容器。从原始表格的 DOM 中提取数据，渲染为卡片行。

### 数据提取

从 `<tbody>` 中每个 `<tr data-row-id="N">` 提取：

| 列索引 | 数据 | 提取方式 |
|--------|------|----------|
| `*-2` | 用户名 | 文本内容，格式 `username (中文名)` |
| `*-6` | 开始时间 | 文本内容，格式 `YYYY-MM-DD HH:MM:SS` |
| `*-7` | 时长 | 文本内容，需解析多种格式 |
| `*-8` | 状态 | `label-success` = 使用中, `label-ignore` = 已结束 |
| `*-9` | 回放按钮 | 提取 `data-record-id` 获取录像 ID |

### 时长解析

支持以下格式，统一转为总秒数：
- `X小时Y分Z秒` → `X*3600 + Y*60 + Z`
- `X分Y秒` → `X*60 + Y`
- `X秒` → `X`
- 带旋转图标的（使用中）：去除 `<i>` 标签后解析文本

### 卡片设计

每张卡片为一行，flex 布局：

```
[头像圆] [用户名]     [时长大字]  [播放按钮]
         [开始时间]
```

- **头像圆**：32px，用户名首字母，渐变背景色（根据用户名 hash 生成）
- **用户名**：主标题 14px，显示中文名为主（括号内的中文名），英文 ID 为辅（副标题或 tooltip）
- **开始时间**：副标题 11px，灰色，格式简化为 `MM-DD HH:MM`
- **时长**：18px 粗体，tabular-nums。颜色规则：≥58min 蓝色 `#0a84ff`，<58min 绿色 `#30d158`
- **播放按钮**：蓝色圆角按钮 `▶ 播放`，点击在新 tab 打开浏览器播放器
- **已缓存标记**：如录像已缓存，按钮旁显示小缓存图标

### "使用中"状态

- 卡片左侧显示绿色脉冲圆点（CSS animation）
- 时长数字带旋转动画或脉冲效果
- 不显示播放按钮（录像尚未结束）
- 显示"使用中"标签替代播放按钮

### 布局

- 单列垂直排列，`gap: 6px`
- 卡片背景白色，圆角 8px，轻阴影
- 原始表格的刷新列表、过滤器按钮保留在卡片列表上方
- 原始表格用 `display: none` 隐藏

### 动态更新兼容

保留现有的 MutationObserver 机制。当原始表格 DOM 变化时（翻页、刷新、新行加入），重新扫描 tbody 并更新卡片列表。

## 2. 播放器快捷键 + 动画

### 快捷键

| 按键 | 功能 | 已有 |
|------|------|------|
| `Space` | 暂停 / 播放 | 是 |
| `←` | 快退 10 秒 | 是 |
| `→` | 快进 10 秒 | 是 |
| `+` / `-` | 变速（循环 1x/2x/4x/8x/16x） | 是 |

所有快捷键已实现，本次增加视觉反馈。

### YouTube 式播放/暂停动画

按空格键时：

1. 画布中央出现半透明圆形图标（64px，`background: rgba(0,0,0,0.65)`，圆角 50%）
2. 图标内容：暂停时显示 ▶，播放时显示 ⏸
3. 动画：0.5s 内从 `opacity: 0.8; scale: 1` → `opacity: 0; scale: 1.5`，使用 CSS animation
4. 动画结束后移除 DOM 元素（或设 `display: none`）

### 快退/快进动画

按左右箭头时：

1. 画布对应侧（左/右）短暂闪烁一个 `«10s` / `10s»` 文字提示
2. 0.3s 后淡出

### 快捷键提示条

- 首次打开播放器时，画布底部居中显示半透明提示条
- 内容：`Space 暂停/播放 | ← → 快退/快进 10s | +/- 变速`
- 3 秒后自动淡出（CSS transition `opacity`）
- 可通过"帮助"菜单再次显示

## 3. 录像缓存

### 存储方式

使用 Cache API。key 格式为 `tp-player-cache` 缓存名下的 URL：`/tp-cache/{rid}/{filename}`。

选择 Cache API 的原因：
- 专为 Request/Response 设计，适合存储大型二进制数据
- 不占用 localStorage 配额
- 支持按 URL 精确查找和删除

### 缓存流程

```
downloader.readFile(filename)
  → 构造 cacheKey = `/tp-cache/${rid}/${filename}`
  → cache.match(cacheKey)
  → 命中: 返回 response.arrayBuffer()
  → 未命中: fetch → clone response → cache.put(cacheKey, clone) → 返回 arrayBuffer
```

对于 `readFileWithProgress`（带进度回调）：缓存命中时直接返回完整 buffer，进度立即跳到 100%。

### 容量管理

- 默认上限：2GB
- 淘汰策略：LRU（Least Recently Used）
- 实现：在 localStorage 中维护 `tp_cache_meta` 对象，记录每个 `{rid}` 的总大小和最后访问时间
- 每次缓存新录像后检查总量，超出上限则删除最久未访问的录像的所有文件
- 可在播放器设置中调整上限或手动清除缓存

### UI 体现

- **列表页**：已缓存的录像卡片上播放按钮旁显示小缓存图标（如 ●）
- **播放器加载**：已缓存显示 "从缓存加载"，未缓存显示正常下载进度百分比
- **侧边栏信息区**：显示缓存状态（已缓存 / 未缓存）和大小

## 4. 顶部菜单栏 + 播放历史

### 菜单栏结构

替换现有的 `#top-bar`（仅显示 meta-info），改为完整菜单栏：

```
[▶ RDP 录屏回放]  [文件] [历史] [帮助]          [用户名 | 日期时间]
```

- 高度：36px
- 样式：frosted glass（保持现有 backdrop-filter 风格）

### 菜单项

**文件菜单（下拉）：**
- 返回列表 — 跳回审计列表页（`/audit/record`）
- 清除缓存 — 清除当前录像或全部缓存

**历史菜单（下拉面板）：**
- 显示最近播放的 20 条录像
- 每条：用户名 + 时长 + 日期
- 点击直接在当前 tab 切换播放（修改 URL + 重新 init，不开新 tab）
- 当前正在播放的录像高亮标记

**帮助菜单（下拉）：**
- 显示快捷键说明

### 播放历史数据

- 存储：localStorage key `tp_play_history`
- 格式：JSON 数组，每条 `{ rid, user, duration, date, timestamp }`
- 最多保存 20 条，新的在前，超出删除最旧的
- 每次打开播放器时自动记录（header 解析完成后写入）

### 播放器内切换录像

从历史列表点击另一条录像时：
1. 更新 `window.__TP_RID` 和 URL 参数
2. 重新调用 `init()` 函数
3. 不刷新页面，不开新 tab

## 5. 右侧笔记侧边栏

### 布局

侧边栏固定在画布右侧，宽度 220px，不可折叠。上下分区：

```
┌─────────────┐
│ 📝 笔记      │  ← 上部，flex: 1，可滚动
│ [快捷标签]    │
│ [文本输入区]  │
│              │
├─────────────┤
│ 📄 信息      │  ← 下部，固定高度
│ 元数据列表    │
└─────────────┘
```

### 笔记区（上部）

- **快捷标签**：三个可点击的 pill 标签
  - 通过（绿色 `#30d158`）
  - 不通过（红色 `#ff453a`）
  - 待定（橙色 `#ff9f0a`）
  - 点击选中一个，再点取消。互斥单选。
- **文本输入区**：`<textarea>`，自适应高度，占据笔记区剩余空间
  - placeholder: "输入对该候选人的评价..."
  - 输入时自动保存（debounce 500ms）

### 信息区（下部）

显示当前录像的元数据，固定高度约 120px：
- 用户：lizh (李志涵)
- 时长：1h 01m 32s
- 开始：2026-03-28 21:13:12
- 分辨率：1920 × 1080
- 远程：Administrator@114.132.204.154
- 缓存：已缓存 (128MB) / 未缓存

### 数据存储

- localStorage key: `tp_player_notes`
- 格式：`{ [rid]: { tag: "pass"|"fail"|"pending"|null, text: "..." } }`
- 每个录像独立存储
- 打开播放器时加载对应 rid 的笔记
- 列表页可读取该数据，在卡片上显示已有评价标签（如果有的话）

## 响应式适配

目标：桌面屏幕（≥1024px 宽），不考虑手机。

- **列表页**：卡片宽度 100%，单列布局，在任何桌面宽度下表现一致
- **播放器**：侧边栏固定 220px，画布区 `flex: 1` 自适应剩余宽度
- **窄屏（1024-1280px）**：侧边栏 220px 仍可正常显示，画布区至少 800px
- **宽屏（≥1920px）**：画布区充分利用空间，侧边栏比例自然缩小

---

## 不在范围内

- 手机/平板适配
- 多人协作笔记（笔记仅本地存储）
- 录像导出/下载
- 视频转码或格式转换
